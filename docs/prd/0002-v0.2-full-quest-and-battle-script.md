# PRD: v0.2.0 完整進關、回合戰鬥腳本與可調解析度（TW / BlueStacks 5）

## Problem Statement

v0.1 只能在禮裝介面點單一 Quest anchor，無法從主畫面自動走到出擊，也無法依回合指定技能、禮裝技能、指令卡與寶具。Operator 若 BlueStacks 視窗非 1920×1080 則無法執行 Run。

v0.2.0 以**一條可重複驗收的 TW 流程**為 tracer：迦勒底之門 → 每日任務 → 打開寶物庫之門（極）→ 選好友 → 選隊伍 → 出擊，並以 YAML 腳本驅動完整戰鬥回合。

## Solution

FGO-AUTO v0.2.0 extends the existing service layer and desktop app:

- **Quest profile**: quest id, navigation script, battle script, party slot, friend support filters
- **NavigationEngine**: declarative `steps[]` (tap_anchor, wait_screen, delay)
- **BattleScriptEngine**: `turns[]` with servant_skill, craft_skill, select_cards, noble_phantasm
- **Display preset pack**: catalog/anchors per `width x height`; tap coordinates normalized 0–1
- **script_version**: `v0` (legacy) vs `v2` (profiles) in Run config

## User Stories

1. As an Operator, I want a Run to start from the main menu and reach deploy for 寶物庫之門（極） without manual navigation.
2. As an Operator, I want to configure which quest profile, party, and friend support to use in YAML or GUI.
3. As an Operator, I want each battle turn scripted (skills, craft, cards, NP) in YAML.
4. As an Operator, I want to set display resolution to match my BlueStacks window and store matching catalog packs.
5. As an Operator, I want Run pause with screenshot when recognition fails, same as v0.1.
6. As an Operator, I want Manual stop and Loop limit to still apply.

## Implementation Decisions

| Area | Decision |
|------|----------|
| First quest flow | 迦勒底之門 → 每日任務 → 寶物庫之門極 → 好友 → 編隊 → 出擊 |
| Battle depth | Full turn: skills, craft, cards, NP in v0.2.0 |
| Resolution | ADR-0005: configurable preset; no auto-scale templates |
| Coordinates | Normalized (x,y) in 0–1 for taps; templates per preset pack |
| UI | Extend GUI for quest profile and resolution (#22); #10 for visual anchor editor (#23) |

## Defaults

- recognition_retries: 5
- Default display_preset: 1920, 1080 (recommended)

## Testing

- pytest with fixtures only in CI
- Manual checklist: `docs/manual-test-v0.2.md` (issue #24)
- No live emulator in CI

## Out of Scope (v0.2.0)

- All quest types (story, event maps) — copy profile template later
- OCR / LLM vision
- Maintenance popups, download dialogs
- Auto-scaling templates across resolutions
